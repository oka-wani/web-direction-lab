# コンテンツ更新方針

## 更新リズム

| コンテンツ | 基本頻度 | 作り方 | 公開条件 |
| --- | --- | --- | --- |
| コラム | 毎日1件 | 記事とショート動画を同じテーマから同時生成 | 人が内容・表現・動画を確認して承認 |
| ニュース | 2日に1件 | 公式発表や一次情報を起点に要約・解説 | 情報源、発表日、事実関係を確認して承認 |
| ナレッジ | 毎日1件＋随時 | 自動テーマ提案、またはユーザーの調査メモから作成 | 出典、実務上の注意、用語、関連ナレッジを確認して承認 |
| 基礎知識 | 初期にまとめて公開 | 学習順と依存関係を先に設計して一括作成 | 全体の重複・不足・導線を横断確認して承認 |

## ナレッジの2つの作成ルート

### 1. 自動テーマルート

1. 未作成テーマと既存記事を照合する
2. 毎日1テーマを選ぶ
3. 一次情報を優先して調査する
4. 本文、用語集、参考サイト、関連ナレッジ、MV案を生成する
5. 下書きプレビューを作る
6. 人が確認・修正して公開する

### 2. 調査メモルート

1. ユーザーが調べた内容を貼り付ける
2. AIが論点、用語、事実、意見を分けて整理する
3. 不足情報を一次情報で補足する
4. 既存記事との重複を確認し、新規記事か追記かを判断する
5. ナレッジ形式に整え、下書きプレビューを作る
6. 人が確認・修正して公開する

## 運用上の原則

- 基礎知識は本数を日割りにせず、学習体験を成立させる最小セットを先に揃える。
- コラムは動画化しやすい「一つの問い・一つの結論」を基本にする。
- ニュースは本数より一次情報と鮮度を優先し、2日に1件を目安にする。
- ナレッジは自動生成と本人の学習記録の両方を蓄積し、後者を独自性の中心にする。
- すべて公開前に人の承認を通し、誤りは既存記事を更新して履歴を残す。
