# 自動投稿フロー

## 目的

毎朝、ナレッジ1件とWebニュース1件の候補を作り、人が5〜10分で確認してから公開する。AIが直接本番公開することはせず、必ず `draft → approved → published` の順に進める。

## 毎朝の流れ

1. 06:00 テーマ候補を収集する
2. 06:10 重複・鮮度・対象読者との関連性を採点する
3. 06:15 ナレッジ1件、Webニュース1件の下書きを生成する
4. 06:30 SEO項目、出典、禁止表現、リンクを機械チェックする
5. 06:35 ショート動画候補を1件選ぶ
6. 07:00 人へ確認依頼を出す
7. 承認後、記事・動画台本・SNS文を公開対象へ移す
8. 公開後、URLと公開時刻を記録する

## 動画にする記事の選び方

- 基本配分はナレッジ70%、Webニュース30%
- 30〜60秒で一つの結論を説明できるものを優先
- 大きな仕様変更やSEOアップデートはニュースを優先
- 出典が一次情報まで確認できないニュースは動画にしない

## ディレクトリ

```text
automation/
  drafts/       AIが生成した未承認データ
  approved/     人が承認した公開待ちデータ
  published/    公開済みの記録
  schemas/      入力データの仕様
content/
  knowledge/    サイトに表示するナレッジ
  news/         サイトに表示するWebニュース
```

## 人が確認する項目

- 結論が明確か
- 誤情報や断定しすぎた表現がないか
- ニュースの一次情報URLが正しいか
- 実務例が読者に役立つか
- タイトルと説明文が内容を正しく表しているか
- 公開して問題ないか
