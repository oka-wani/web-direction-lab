# WGL プラットフォーム設計

## コンセプト

**Webで成果を出すための知識・ツール・サービスを提供するプラットフォーム。**

Web Growth Labは学習メディアだけを目指さない。Web制作・SEO・AI・業務改善の情報を発信しながら、実務ツール、テンプレート、Webサイト診断・改善・制作・運用支援を提供する。

訪問者の体験を次の3段階でつなぐ。

1. 学ぶ
2. 使う
3. 相談する

## ターゲット

- 中小企業
- 個人事業主
- 社内Web担当者
- Webディレクター
- 制作担当者

主な課題は、問い合わせ不足、SEO、改善点の不明確さ、一人Web担当、AI導入、業務効率化とする。

## 3つの収益エンジン

### 1. Web改善・コンサルティング

短期的な売上を作る受注事業。簡易診断、改善提案、SEO、Web制作、CMS、AI導入、運用・保守を提供する。

### 2. メディア

検索・SNSからの集客と信頼形成を担う。Webガイド、ナレッジ、コラム、ニュースを運営し、将来的にアフィリエイト、広告、テンプレート販売へつなげる。

### 3. プロダクト

繰り返し販売できる事業。初期は「ツール」として無料ツール、テンプレート、AIプロンプトを公開し、商品が増えた段階でSaaS、AIエージェントを含む「プロダクト」へ独立させる。

## コンテンツの役割

| 種別 | 役割 | 主な導線 |
|---|---|---|
| Webガイド | 体系的な理解 | ナレッジ、ツール、相談 |
| ナレッジ | 検索流入・専門性 | 関連サービス |
| コラム | 考え方・実践・信頼形成 | ツール、サービス |
| ニュース | 最新性・継続運営の証明 | 関連ナレッジ、サービス |
| ツール | 利用体験・見込み顧客化 | テンプレート、相談 |
| サービス | 問い合わせへの転換 | お問い合わせ |

記事下のCTAはテーマに合わせて出し分ける。SEO記事はSEO改善、CMS記事はCMS導入・運用設計、AI記事はAI導入支援、UI・UX・アクセシビリティ記事はWebサイト改善へ接続する。

## 初期商品

入口商品はWebサイト簡易診断とする。URLと悩みを受け取り、SEO、UI・UX、導線、コンテンツ、アクセシビリティ、技術品質、運用を確認し、良い点、改善点、優先順位、次のアクションを返す。

初期価格の目安は5,000円から。改善提案レポート、部分改修、制作、継続運用へ段階的につなげる。

## 開発ロードマップ

### Phase 1：受注基盤

- サービス、料金、進め方、FAQ、お問い合わせ
- TOPを受注型へ変更
- 全記事へ内容別CTAを設置
- 簡易診断と改善レポートのテンプレート化

### Phase 2：メディア・SEO

- SEO技術基盤と構造化データ
- GA4、Search Console、GTM、Clarity
- 記事からサービス、フォーム、完了までの計測
- 集客、信頼形成、受注直前の3種類のコンテンツを拡充

### Phase 3：ツール・プロダクト

- 無料ツールとテンプレート
- AIプロンプト
- 買い切り商品
- SaaS、AIエージェント

### Phase 4：継続改善の自動化

- 毎日：異常検知、Search Console・GA4・エラー情報の収集
- 毎週：CTR、順位、サービス遷移率、改善候補の分析とPR作成
- 毎月：問い合わせ経路、サービス別成果、翌月計画の分析

## 運用原則

- 実在しない事例や未完成機能を、提供中のように見せない
- AIの出力は人が確認する
- 記事量より、検索から問い合わせまでの導線と計測を優先する
- サービスの受け皿を整えてから集客コンテンツを増やす
- 最終的に「検索で見つかる → 信頼される → 使う・相談する → 改善する → 継続支援」までを半自動で回す
